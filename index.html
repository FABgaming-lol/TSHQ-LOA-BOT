2. Warning System New

Tracks user strikes. At 3 warnings, an automatic alert is sent to the staff channel.

Issue a Warning

/warn [user] [reason]
  • Logs the warning in the database.
  • DMs the user telling them why.
  • Adds +1 to their strike count.

Manage Warnings

View History

/check_warnings [user]

Shows dates, mods, and reasons.

Remove One / Reset

/unwarn [user]
/clear_warnings [user]

3. Moderation Tools

All actions are logged in the Mod Logs channel.

Ban User (Supports User IDs)

/ban [user] [reason]

Tip: You can paste a User ID to ban someone who isn't in the server.

Kick User

/kick [user] [reason]

Timeout / Mute

/timeout [user] [minutes] [reason]

Example: /timeout @User 60 Spamming (1 hour)

Purge Chat

/purge [amount]

Deletes messages instantly.

Auto-Moderation

The bot passively watches chat for Blacklisted Words.

Action

Automatically deletes the message and warns the user in chat.

Immunity

Staff members can speak freely without triggering the filter.

5. Leave of Absence

Managers Only

Start LOA

/loa [user] [duration] [reason]
7d (Days) 2w (Weeks) 1m (Months)

The bot will automatically remove the LOA role when time expires.

End Early

/endloa [user]

Use this if a staff member returns before their scheduled date.

7. Logging Channels

The bot uses nice colored Embeds to track everything. Logs are split into these channels:

Mod Logs

Bans, Kicks, Timeouts, Purges, and Warnings.

Ticket Logs

Ticket Transcripts (Text files) and Open/Close alerts.

Server Logs

Auto-Mod triggers, Deleted/Edited messages, Joins/Leaves, and LOA updates.

Troubleshooting

"Interaction Failed"

The bot is likely restarting. Please wait 30 seconds and try again.

"Permission Denied"

You do not have the required Manager or Mod role to use this command.

Announcement Failed

The bot does not have permission to "Send Messages" in the destination channel. Check the channel's specific permissions settings.

Commands Missing?

Run the !sync command in any chat and restart your Discord app.
