3. Warning System

3-Strike Rule: Always warn a user before kicking/banning (unless it's severe).

Core Commands

/warn [user] [reason]

The bot sends a DM to the user explaining why they were warned.

/check_warnings [user]

See all past warnings for a specific user.

Management

Remove Recent

/unwarn [user]

Reset Everything

/clear_warnings [user]

üö® Automatic Alert

If a user reaches 3 Warnings, the bot will automatically ping the Managers in the staff channel to review the user.

4. Moderation Tools

Server Access

/kick [user] [reason]
/ban [user] [reason]

Tip: Paste a User ID (e.g., 987654...) to ban users not in the server.

Chat Management

/timeout [user] [minutes] [reason]
/purge [amount]

Example: /purge 10 deletes the last 10 messages.

Auto-Mod

The bot automatically deletes messages containing words like:

"scam" "cheat" "discord.gg"

Note: Staff members are immune to this filter so you can discuss these topics freely.

5. Handling Tickets

When a user opens a ticket, a private channel is created automatically.

Guest Access

/ticket_add [user]

If you need to add an expert or manager to the ticket.

/ticket_remove [user]

Closing Procedure

  • 1

    The bot saves the entire chat history as a file.

  • 2

    Sends the transcript to the Ticket Logs channel.

  • 3

    Deletes the ticket channel instantly.

6. Leave of Absence Managers Only

Use this to track staff breaks and vacations.

Start LOA

/loa [user] [duration] [reason]
7d (Days) 2w (Weeks) 1m (Months)

The bot automatically removes the LOA role when the time expires.

End Early

/endloa [user]

Use this if staff return before their scheduled date.
