2. Moderation & Auto-Mod

Moderators Managers

Manual Commands

Kick User

!kick @User [Reason]

Ban User (Supports IDs)

!ban @User [Reason]

You can ban a User ID even if they are not in the server.

Timeout / Mute

!timeout @User [Minutes] [Reason]

Example: !timeout @User 60 Spamming (1 hour mute)

Purge Messages

!purge [Amount]

Example: !purge 20

Auto-Moderation

The bot automatically watches chat for Blacklisted Words.

Action

Deletes the message immediately and warns the user via DM/Logs.

Logs

Sends a detailed alert to the Server Logs channel.

Immunity

Moderators and Managers can say these words without punishment.

3. Ticket System

Transcripts are now saved automatically

Setup Admin Only

!setup_tickets

(Spawns the "Open Ticket" panel in current channel)

Managing Tickets

!ticket_add @User
!ticket_remove @User

Closing a Ticket

The bot will generate a transcript file and send it to the Ticket Logs channel before deleting the ticket.

4. Leave of Absence (LOA)

Managers Only

Start a Leave

!loa @User [Duration] [Reason]
7d (Days) 2w (Weeks) 1m (Months)

End a Leave Early

!endloa @User

The bot automatically removes the role when time expires. No manual action required.

5. Logging Locations

The bot now splits logs into specialized channels for maximum organization.

Moderation Logs

Detailed reports of Bans, Kicks, Timeouts, and channel Purges.

Ticket Logs

Text-based transcripts, and all Open/Close alerts.

Server Logs

Auto-Mod warnings, message edits, join/leave events, and LOA updates.

Troubleshooting

"Permission Denied"?

Announcements/LOA

These commands are strictly restricted to the Managers role only.

Moderation Action

Ensure the Bot's Role is physically higher in the role hierarchy than the target user.

Auto-Mod didn't delete my message?

All staff members are immune to the blacklisted word filter to prevent accidental mutes. To test, use a non-staff test account.
